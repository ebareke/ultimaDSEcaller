# Citing ultimaDSEcaller

If you use **ultimaDSEcaller** (the tool, its Nextflow pipeline, or its
container images) in your research, please cite it. A machine-readable
[`CITATION.cff`](CITATION.cff) is also provided, which powers the
**“Cite this repository”** button on GitHub.

## Software

> Bareke, E., Bareke, E., & Bareke, C. (2026). *ultimaDSEcaller:
> Ultimate Differential Splicing Event caller for short- and long-read
> RNA-seq* (Version 1.0.0) [Computer software].
> https://github.com/ebareke/ultimaDSEcaller

## Please also cite the tools ultimaDSEcaller builds on

Depending on which parts of the workflow you use, the following upstream
tools should be cited:

- **STAR** (short-read alignment): Dobin, A., et al. (2013). STAR: ultrafast
  universal RNA-seq aligner. *Bioinformatics*, 29(1), 15–21.
- **minimap2** (long-read alignment): Li, H. (2018). Minimap2: pairwise
  alignment for nucleotide sequences. *Bioinformatics*, 34(18), 3094–3100.
- **HISAT2** (alternative short-read alignment): Kim, D., et al. (2019).
  Graph-based genome alignment and genotyping with HISAT2 and
  HISAT-genotype. *Nature Biotechnology*, 37, 907–915.
- **SAMtools / HTSlib** (BAM/CRAM handling): Danecek, P., et al. (2021).
  Twelve years of SAMtools and BCFtools. *GigaScience*, 10(2), giab008.
- **Nextflow** (workflow engine, if used): Di Tommaso, P., et al. (2017).
  Nextflow enables reproducible computational workflows. *Nature
  Biotechnology*, 35, 316–319.

## Versioning

Please cite the exact version you used (`ultimaDSEcaller --version`). Each
tagged release is archived on GitHub Releases with reproducible binaries and
container images.
